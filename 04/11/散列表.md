## 散列（hash）表
### 概念
- 散列函数 H(key) = addr 或者 Hash(key) = addr  
addr:``数组下标`` ``索引`` ``内存地址``

- 冲突 
- 同义词
```
冲突：
H(key1) = addr_1
H(key2) = addr_1

key1,key2为同义词
```
```
Q：良好的散列函数有什么性质？
1. 尽可能减少冲突的发生
2. 冲突不可避免  设计有效的冲突处理方法
```
- 散列表(哈希表) ：根据关键字进行直接访问的数据结构
``理想情况时间复杂度O(1)``
### 散列函数的构造方法
- 构造的时候应注意的内容
```
1.定义域包含所有关键字key 值域的取值范围取决于散列表大小
2.hash出来的地址均匀分布 减少冲突
3.函数简单 减少计算出地址的时间
```

1. 直接定址法
```
H(key) = key;
H(key) = a key + b;
本质一次函数

好处：
1.计算最简单
2.不会产生冲突

适用：
1.关键字分布基本连续     不连续则可能会造成空间浪费
```
2. 除留余数法
```
m:散列表长
H(key) = key % p   p:不超过m且非常接近m的质数

最简单最常用
```
3. 数字分析法
```
10012001
10012002
10012003 
选取最后三位进行hash
```
4. 平方取中法

```
key = 1234
key^2 =  1522756 
取中间两位27
H(1234) =  27

尽量居中截取
表长 m	需要位数
10	    1位
100	    2位
1000	3位
```

### 散列表查找以及性能
1. ASL衡量散列表查找效率的核心指标 由于冲突存在 查找过程仍需要将给定值与表中关键字对比
```
H(key) = key % 10

key1 = 21 → H(21) = 1  
key2 = 31 → H(31) = 1

拉链法
地址1 → [21] → [31]

查找
key = 31
H(31) = 1
比较：
地址1 → [21] → [31]

```
2. 散列表的查找效率取决于三个因素：
- 散列函数
- 冲突处理方法
- 装填因子 α = 已装填m/散列表长度n
